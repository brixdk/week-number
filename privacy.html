Privacy Policy for Week Number

Last updated: November 14, 2025

1. Overview

Week Number is a Chrome extension designed to integrate with your Google Calendar to provide a convenient calendar overview with week numbers. This Privacy Policy explains how the extension handles your data and your privacy rights.

2. Information We Collect

2.1 Google User Data

When you connect your Google Calendar, the extension requests and accesses the following Google user data through Google's OAuth 2.0 authorization:

Calendar Data (via calendar.readonly scope):

  • Calendar event details: event titles, descriptions, start/end times, locations
  • Event participants: attendee email addresses, names, and response status
  • Event metadata: event colors, HTML links to events, all-day event indicators
  • Video conference links (Google Meet/Hangout links)
  • Your calendar settings: timezone, locale preferences, time format (12/24 hour), date format

Profile Data (via userinfo.email scope):

  • Your email address
  • Your name (if available)

2.2 Local Extension Data

The extension stores the following data locally in your browser:

  • Your authentication token (OAuth access token)
  • User preferences: theme (light/dark/system), week start day (Sunday/Monday), sync preferences
  • Cached calendar events (to improve performance and reduce API calls)
  • Timestamp of last calendar synchronization

3. How We Use Your Information

3.1 Data Usage

The extension uses your Google user data exclusively for the following purposes:

  • Display week numbers: Calculate and display the current week number in the browser extension badge
  • Show calendar overview: Display your calendar events in a monthly view within the extension popup
  • Sync preferences: Match your Google Calendar settings (timezone, time format) for consistent display
  • Event details: Show event information including time, location, attendees, and video conference links
  • Performance optimization: Cache calendar data locally to reduce repeated API requests and improve loading speed

3.2 Data Processing

All data processing occurs locally within your Chrome browser. The extension:

  • Fetches calendar data directly from Google's servers to your browser
  • Processes and displays the data within the extension interface
  • Caches data temporarily in your browser's local storage
  • Does NOT send your data to any third-party servers
  • Does NOT share your data with any external services

4. Data Storage and Retention

4.1 Local Storage

All Google user data is stored exclusively in your browser using Chrome's local storage APIs:

  • OAuth tokens: Stored securely using Chrome Identity API
  • Calendar events: Cached temporarily; automatically refreshes every 30 minutes
  • User preferences: Stored until you change them or disconnect the extension

4.2 Data Retention

  • Calendar data is cached locally and refreshed periodically
  • All data is automatically cleared when you disconnect your Google Calendar
  • Data remains on your device only; no server-side storage exists
  • Uninstalling the extension removes all stored data

5. Data Sharing and Transmission

Week Number does NOT:

  • Share your Google user data with any third parties
  • Sell or monetize your data
  • Transmit your data to any external servers (except direct communication with Google's APIs)
  • Use your data for advertising or analytics

The only data transmission occurs between:

  • Your browser and Google's Calendar API (to fetch calendar data)
  • Your browser and Google's OAuth servers (for authentication)

6. Data Security

Week Number implements the following security measures:

  • Uses Google's official OAuth 2.0 authentication flow (industry standard)
  • Never handles or stores your Google password
  • Stores authentication tokens using Chrome's secure storage APIs
  • Requests only read-only access to your calendar (cannot modify or delete events)
  • Processes all data locally within your browser's secure environment

7. Your Privacy Rights

You have complete control over your data:

  • Access: View all stored data through Chrome's developer tools
  • Revoke access: Disconnect Google Calendar at any time through extension settings
  • Delete data: All data is cleared when you disconnect or uninstall
  • Control sync: Enable or disable calendar synchronization in settings

8. Compliance with Google API Services User Data Policy

This extension complies with the Google API Services User Data Policy, including the Limited Use requirements. Specifically:

  • Week Number only uses Google user data to provide and improve user-facing features
  • Data usage is limited to practices disclosed in this privacy policy
  • The extension does NOT transfer Google user data to third parties (except as necessary to provide the extension functionality or as required by law)
  • The extension does NOT use or transfer Google user data for serving ads
  • The extension does NOT use or transfer Google user data to determine creditworthiness or for lending purposes

9. Third-Party Services

The extension uses:

  • Google Calendar API: For accessing your calendar data (subject to Google APIs Terms of Service)
  • Google OAuth 2.0: For secure authentication (managed by Google)

No other third-party services are used.

10. Changes to this Policy

This Privacy Policy may be updated periodically to reflect changes in:

  • Extension features
  • Legal requirements
  • Google API policies

Updates will be published on this page with a new "Last updated" date. Significant changes will be communicated through the Chrome Web Store listing.
